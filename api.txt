========================================
--- CLASS MANAGEMENT CONTROLLER (api/v1/class) ---
========================================

Class Initialization:
1. POST /api/v1/class
   - Desc: Create a new class instance (ClassSemester linkage).
   - Roles: admin, manager
   - Request Body:
     {
       "classCode": "SE1234",
       "subjectId": "guid",
       "semesterId": "guid",
       "teacherId": "guid (optional)"
     }
   - Response Body (201 Created):
      { "code": 200, "message": "Class instance created successfully", "data": { "classId": "guid", "instanceId": "guid", "classCode": "SE1234" } }

2. GET /api/v1/class
   - Desc: View all classes (Admin/Manager sees all, Teacher sees their assigned classes).
   - Roles: admin, manager, teacher
   - Query Params: ?semesterId=...&subjectId=...&search=...
   - Response Body (200 OK - ApiResponse):
      { "code": 200, "message": "Classes fetched successfully", "data": { "items": [ /* ClassResponse */ ], "totalCount": 1 } }

3. GET /api/v1/class/{id}
   - Desc: Get class instance details by Class ID. Returns nested ClassInstanceInfo details.
   - Roles: admin, manager, teacher
   - Response Body (200 OK - ApiResponse): { "code": 200, "message": "Class details fetched successfully", "data": { /* ClassResponse */ } }

Student Management:
4. GET /api/v1/class/{classSemesterId}/students
   - Desc: Get all active/inactive students in a specific class instance.
   - Roles: admin, manager, teacher
   - Response Body (200 OK - ApiResponse):
      { "code": 200, "message": "Students fetched successfully", "data": [ /* ClassMemberResponse */ ] }

4.1. GET /api/v1/class/export/template
     - Desc: Export Excel template for importing students into a class.
     - Roles: admin, manager
     - Response: Binary Excel File stream (`.xlsx`).

5. POST /api/v1/class/{classSemesterId}/students/import
   - Desc: Import student mapping via Excel upload.
   - Roles: admin, manager, teacher
   - Content-Type: multipart/form-data
   - Response Body (200 OK): { "totalProcessed": 10, "successCount": 10, "failedCount": 0, "errors": [] }

6. GET /api/v1/class/{classSemesterId}/students/export
   - Desc: Export the list of students in the class to an Excel worksheet.
   - Roles: admin, manager, teacher
   - Response: Binary Excel File stream (`.xlsx`).

7. POST /api/v1/class/{classSemesterId}/students/manual
   - Desc: Manually add a student using RollNumber or MemberCode.
   - Roles: admin, manager, teacher
   - Request Body: { "rollNumber": "HE123", "memberCode": "SE1" }
   - Response Body (200 OK): { "message": "Student added successfully.", "studentId": "guid" }

8. PUT /api/v1/class/{classSemesterId}/students/{studentId}
   - Desc: Update learning status (IsActive) for a student in class.
   - Roles: admin, manager, teacher
   - Request Body: { "isActive": false }
   - Response Body (200 OK): { "message": "Student status updated successfully." }

9. DELETE /api/v1/class/{classSemesterId}/students/{studentId}
   - Desc: Completely remove an enrolled student from the class instance.
   - Roles: admin, manager, teacher
   - Response Body (200 OK): { "message": "Student removed from class successfully." }

Class Ranking:
10. GET /api/v1/class/{classSemesterId}/ranking
    - Desc: Get overall ranking overview based on total score and solved count across all assignments.
    - Roles: admin, manager, teacher, student
    - Response Body (200 OK - ApiResponse):
      { "code": 200, "message": "Ranking fetched successfully", "data": [ /* ClassRankingEntry */ ] }

Invite Codes:
11. POST /api/v1/class/{classSemesterId}/invite-code
    - Desc: Generates a temporary invite code string to allow self-enrollment. (Max ~15 mins).
    - Roles: admin, manager, teacher
    - Request Body: { "minutesValid": 15 }
    - Response Body (200 OK): { "inviteCode": "string", "expiresAt": "datetime" }

12. DELETE /api/v1/class/{classSemesterId}/invite-code
    - Desc: Invalidates and disables any active invite code.
    - Roles: admin, manager, teacher
    - Response Body (200 OK): { "message": "Invite code cancelled." }

Class Contests:
13. GET /api/v1/class/{classSemesterId}/contests
    - Desc: View all organized class contests.
    - Response Body (200 OK - ApiResponse): List of ClassContestSummaryResponse

14. POST /api/v1/class/{classSemesterId}/contests
    - Desc: Teacher directly creates a contest scoped to the class.
    - Roles: admin, manager, teacher
    - Request Body: CreateClassContestRequest (Title, Problems array, Start/End times)
    - Response Body (201 Created): { "message": "Contest created successfully." }

15. GET /api/v1/class/{classSemesterId}/contests/{contestId}
    - Desc: View details of a specific contest (including TimeLimit and joined status).
    - Response Body (200 OK - ApiResponse): ClassContestResponse object

16. PUT /api/v1/class/{classSemesterId}/contests/{contestId}/extend
    - Desc: Extends the deadline of an active contest.
    - Roles: admin, manager, teacher
    - Request Body: { "newEndAt": "2026-..." }
    - Response Body (200 OK): { "message": "Contest time extended successfully." }

17. POST /api/v1/class/{classSemesterId}/contests/{contestId}/join
    - Desc: Students sign up and join the active class contest.
    - Response Body (200 OK): { "message": "Joined contest successfully." }


========================================
--- CLASS SLOT CONTROLLER (api/v1/class-instance/{instanceId}/slots) ---
========================================

Slot Management:
1. GET  /api/v1/class-instance/{instanceId}/slots
   - Desc: List all slots for a class instance.
   - Response Body (200 OK - ApiResponse):
     { "code": 200, "message": "Slots fetched successfully", "data": [ { /* SlotResponse */ } ] }

2. POST /api/v1/class-instance/{instanceId}/slots
   - Desc: Create a new assignment/slot. Problems are OPTIONAL at creation time.
   - Roles: admin, manager, teacher
   - Request Body:
     {
       "slotNo": 1,
       "title": "Lab 01",
       "mode": "problemset",
       "problems": [ { "problemId": "guid", "ordinal": 1, "points": 100, "isRequired": true } ]
     }
   - Response Body (201 Created):
     { "message": "Assignment created successfully.", "id": "guid", "slotNo": 1 }

3. PUT  /api/v1/class-instance/{instanceId}/slots/{slotId}
   - Desc: Update slot details (title, description, rules, time, publish). Does NOT touch problems.
   - Roles: admin, manager, teacher
   - Response Body (200 OK): { "message": "Slot updated successfully.", "id": "guid" }

4. DELETE /api/v1/class-instance/{instanceId}/slots/{slotId}
   - Desc: Delete a slot and all associated ClassSlotProblems.
   - Roles: admin, manager, teacher
   - Response Body (200 OK): { "message": "Slot deleted successfully." }

Time & Access:
5. PUT  /api/v1/class-instance/{instanceId}/slots/{slotId}/due-date
   - Desc: Set or change the due date (and optionally close date) of a slot.
   - Roles: admin, manager, teacher
   - Request Body: { "dueAt": "2026-04-07T23:59:59Z", "closeAt": "2026-04-08T23:59:59Z" }
   - Response Body (200 OK): { "message": "Due date updated successfully." }

6. PUT  /api/v1/class-instance/{instanceId}/slots/{slotId}/publish
   - Desc: Toggle publish/unpublish a slot.
   - Roles: admin, manager, teacher
   - Response Body (200 OK): { "message": "Slot published." | "Slot unpublished.", "isPublished": true | false }

Problem Management (CRUD):
7. POST /api/v1/class-instance/{instanceId}/slots/{slotId}/problems
   - Desc: Add one or more problems to a slot. Skips duplicates automatically.
   - Roles: admin, manager, teacher
   - Request Body: [ { "problemId": "guid", "ordinal": 1, "points": 100, "isRequired": true } ]
   - Response Body (200 OK): { "message": "1 problem(s) added to slot.", "added": 1 }

8. PUT /api/v1/class-instance/{instanceId}/slots/{slotId}/problems
   - Desc: Update properties (Ordinal, Points, IsRequired) of problems already added to a slot.
   - Roles: admin, manager, teacher
   - Request Body: [ { "problemId": "guid", "ordinal": 1, "points": 80, "isRequired": true } ]
   - Response Body (200 OK): { "message": "1 problem(s) updated successfully.", "updated": 1 }

9. DELETE /api/v1/class-instance/{instanceId}/slots/{slotId}/problems
   - Desc: Remove one or more problems from a slot by ProblemId.
   - Roles: admin, manager, teacher
   - Request Body: [ "problem-guid-1", "problem-guid-2" ]
   - Response Body (200 OK): { "message": "2 problem(s) removed from slot.", "removed": 2 }

Scoring & Submissions:
10. GET  /api/v1/class-instance/{instanceId}/slots/{slotId}/scores
    - Desc: Get total and individual problem score overview of all students in this slot.
    - Roles: admin, manager, teacher
    - Response Body (200 OK - ApiResponse):
      {
        "code": 200, "message": "Scores fetched successfully",
        "data": [
          {
            "userId": "guid",
            "displayName": "John Doe",
            "problemScores": [
              { "problemId": "guid", "verdictCode": "ac", "score": 100.0, "attempts": 3 }
            ],
            "totalScore": 100.0,
            "solvedCount": 1
          }
        ]
      }

11. GET /api/v1/class-instance/{instanceId}/slots/{slotId}/submissions/{userId}
    - Desc: View detailed submissions of a specific student for problems in this slot.
    - Roles: admin, manager, teacher
    - Response Body (200 OK - ApiResponse):
      {
        "code": 200, "message": "Submissions fetched successfully",
        "data": [
          {
            "submissionId": "guid",
            "problemTitle": "Two Sum",
            "verdictCode": "ac",
            "finalScore": 100.0,
            "results": [
              { "statusCode": "ac", "runtimeMs": 15, "memoryKb": 4096 }
            ]
          }
        ]
      }
